# AoC Initialization and Run Tool

## Overview
aoc is a CLI tool for managing Advent of Code solutions across multiple languages. It has prebuilt templates for several languages, and supports a generic command set over those languages.
The system is constructed in BASH so should be compatible for any computer capable of doing AoC.

## Directory Structure
```
.
├── aoc                           # Main dispatcher script
├── aoc-scripts/
│   ├── aoc-init                 # Initialize day structure
│   ├── aoc-fetch                # Fetch puzzle input
│   ├── aoc-zig-functions        # Zig-specific functions
│   └── aoc-ruby-functions       # Ruby-specific functions
│   └── aoc-...-functions        # Future
├── templates/
│   ├── template-zig/            # Zig template directory
│   └── template-ruby/           # Ruby template
├── .aocrc                       # Config file (language, year)
├── .env                         # Session cookie (AOC_SESSION)
└── day01-zig/                   # Generated day directories
    ├── input/
    │   └── input.txt
    └── src/
```

## Configuration Files

### .aocrc
```bash
AOC_YEAR=${AOC_YEAR:=2025}
AOC_LANGUAGE=${AOC_LANGUAGE:=zig}
```

### .env
```bash
AOC_SESSION=your_session_cookie_here
```

### .aoc_current
Autogenerated, remove to remove cached settings

# Command Line
```
aoc init --day <Day Number> [--lang <Language> --force]
  Initializes the day specified with the appropriate language template
  This command will set the 'current' day, allowing other commands to omit the switch
  This command will set the 'current' language, allowing other commands to omit the switch
  --day <Day Number> day
  --lang specifies a language otherwise the default (<current default>) is used from .aocrc (or Python)
  --force if this day already is initialized, purge and re-initialize THIS IS DESTRUCTIVE

aoc fetch [--day <Day Number> --force --wait]
  Fetch the desired day AoC input.
  --day <Day Number> day
  --force if this input is already downloaded, force redownload
  --wait if before the contest open, will wait until the next start time

aoc run|test [--day <Day Number> --lang <Language> --partA --partB] [-- [Additional Args]]
  Run or test a specified day/language for a given part (or both if not specified)
  This command will set the 'current' day, allowing other commands to omit the switch
  This command will set the 'current' language, allowing other commands to omit the switch
  --day <Day Number> day
  --lang specifies a language otherwise the default (<current default>) is used from .aocrc (or Python)
  --partA run/test partA
  --partB run/test partB
  -- Ends option parsing and passes any additional args to the program
```

## Usage Examples
```bash
# Initialize a new day
./aoc init --day 1 # Creates day01-zig/ and caches AOC_DAY=1 AOC_LANGUAGE=Python

# Fetch input
./aoc fetch --day 1
# Or use cached day:
./aoc fetch

# Fetch with wait (if puzzle not released yet)
./aoc fetch --day 5 --wait

# Run solution
./aoc run --day 1 --partA
./aoc run --day 1 --partB
./aoc run --day 1  # Runs both parts, all of the above cache AOC_DAY=1
./aoc run  # Uses AOC_DAY

# Test solution
./aoc test --day 1 --partA # Tests day 1 partA and caches AOC_DAY=1
./aoc test  # Uses AOC_DAY, runs all tests

# Override language
./aoc init --day 2 --lang python
```
